Zero Downtime

Komponen Nodes ada 2:

  1. Master node
  2. Worker

Master Node

  1. API Server : Jembatan penghubung semua service komponen yang ada
  2. Scheduler : Melakukan penjadwalan ke nodes mana saja pod(s) akan ditempatkan
  3. Controller Manager : Mengontrol replikasi, handling node failures, dll yang berhubungan dengan cluster-nya
  4. Etcd : Menyimpan informasi key-value object dari delpoyment, pod, service

Worker Node

  1. Container runtime : Engine yang jalanin container-nya (docker, rkt, cri, dll)
  2. Kubelet : Agent yang ada di worker nodes untuk komunikasi ke K8S API nya
  3. Kube-proxy : Untuk load balance secara network traffic nya
